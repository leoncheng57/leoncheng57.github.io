# Vite Migration Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Next.js with plain Vite + React on this single-page static portfolio site.

**Architecture:** Clean-slate approach — scaffold a new Vite entry point (`index.html`, `src/main.tsx`, `src/App.tsx`), move existing components and styles into `src/`, fix `@value` CSS incompatibilities, then delete all Next.js artifacts. The site continues to deploy to GitHub Pages via the existing `docs/` output folder.

**Tech Stack:** Vite 6, @vitejs/plugin-react, React 19, TypeScript 5, CSS Modules

**Spec:** `docs/superpowers/specs/2026-03-15-vite-migration-design.md`

**Files:**

This is Vite's HTML entry point. It replaces Next.js's implicit HTML shell. The `<head>` block takes over what `next/head` handled in `pages/index.tsx`, and the GA scripts replace the `next/script` tags.

- [ ] **Step 1: Create the file**

```html

    <meta name="description" content="Generated by create next app" />
    <link rel="icon" href="/favicon.ico" />
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
    <!-- Global site tag (gtag.js) - Google Analytics -->
    <script async src="https://www.googletagmanager.com/gtag/js?id=G-5MLNJQ7789"></script>
    <script>
      window.dataLayer = window.dataLayer || [];
      function gtag(){dataLayer.push(arguments);}
      gtag('js', new Date());
      gtag('config', 'G-5MLNJQ7789');
    </script>

</html>
```

### Task 9: Fix @value CSS incompatibility

**Files:**
- Modify: `src/styles/colors.css`
- Modify: `src/styles/globals.css`
- Modify: `src/components/headline/headline.module.css`

Vite does not support the `@value` directive from `postcss-modules-values`. Three files use it. The fix converts `@value blueEmphasis` to a standard CSS custom property (`--blue-emphasis`).

- [ ] **Step 1: Rewrite src/styles/colors.css**

Replace the entire file:

```css
:root {
  --blue-emphasis: #0575c7;
}
```

- [ ] **Step 2: Import colors.css from globals.css**

Add this line at the top of `src/styles/globals.css`:

```css
@import './colors.css';
```

After the addition, `globals.css` should start with:
```css
@import './colors.css';

html,
body {
  ...
```

- [ ] **Step 3: Fix headline.module.css**

In `src/components/headline/headline.module.css`, remove the two `@value` lines at the top and replace `blueEmphasis` with `var(--blue-emphasis)`.

The file should go from:
```css
@value colors: "../../styles/colors.css";
@value blueEmphasis from colors;
...
.summary > .summaryHighlight {
  color: blueEmphasis;
  ...
}
```

To (remove only the two @value lines and update the color reference):
```css
.container {
  ...
}
...
.summary > .summaryHighlight {
  color: var(--blue-emphasis);
  font-weight: 500;
}
```

- [ ] **Step 4: Commit**

```bash
git add src/styles/colors.css src/styles/globals.css src/components/headline/headline.module.css
git commit -m "fix: convert @value CSS to CSS custom properties for Vite compatibility"
```
